## File Format
All images are in SVG (Scalable Vector Graphics) format for:
- Scalability without quality loss
- Lightweight file sizes
- Easy customization
- Better performance on all devices

## Usage
Images are referenced in `insights.html` and displayed in blog card components with the following structure:
```html
<img src="assets/blog-1.svg" alt="Description" class="blog-image">
```

## Customization
To modify SVG files:
1. Open the .svg file in a text editor or design software (Figvox, Illustrator, Inkscape)
2. Edit the gradients, text, or shapes as needed
3. Save the file
4. Changes will be reflected immediately in the browser

## Adding New Assets
When adding new blog cards:
1. Create new SVG file (blog-7.svg, blog-8.svg, etc.)
2. Name it consistently following the naming convention
3. Update the insights.html file to reference the new image
4. Add appropriate category and styling
